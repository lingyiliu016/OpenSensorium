# T04 · 感知事件总线（统一时间戳、跨模态对齐）

- 里程碑：M0
- 依赖：T00
- PRD 对应：FR-1.5、§6.2 实时数据面
- 状态：⬜ 未开始

## 目标
建立 Afferent 的中枢管道：所有感知层输出（视频帧、音频片段、文本、Delegate 回流…）都变成**带统一时间戳的 `PerceptionEvent`**，认知核订阅消费。同时区分**控制面**（切源/停止）与**行动面**。

## 交付物
- [ ] `src/opensensorium/bus/event_bus.py`：异步发布/订阅（asyncio），支持多订阅者、背压/丢帧策略。
- [ ] 每个事件带 `source_id` + 单调时间戳 + 序号，为未来多路预留（§5.2）。
- [ ] `src/opensensorium/bus/control_plane.py`：独立控制指令通道（切源、停止），与感知事件分离。
- [ ] 背压：视频按显著性动态采样的钩子（静止降帧/变化升帧，具体策略在 T12）。
- [ ] 测试：`tests/test_event_bus.py`（顺序、时间戳单调、背压丢帧、控制面与感知面隔离）。

## 实现要点
- 用 asyncio Queue + 订阅者模式；慢订阅者不得阻塞采集（NFR 实时性）。
- 时间戳对齐口径统一走 T00 的时钟工具，供 MiniCPM-o TDM 交织对齐。

## 分工（Agent 做什么 / 人做什么）
- **Agent**：实现总线、控制面、背压策略 + 单元测试，全部可离线自测。
- **人**：审背压/丢帧策略是否符合"实时不阻塞采集"的预期。

## 验收标准（可验证）
- [ ] `tests/test_event_bus.py` 全绿：多订阅者收到同序事件；时间戳单调递增。
- [ ] 背压测试：高频发布下慢订阅者按策略丢帧而非阻塞发布方。
- [ ] 控制指令走独立通道，不与 `PerceptionEvent` 混流（断言隔离）。
