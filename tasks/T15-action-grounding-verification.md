# T15 · 动作坐标接地 + 执行后校验（GUI 闭环可靠性）

- 里程碑：M2（看家能力收口）
- 依赖：T13、T14
- PRD 对应：FR-3.6、§12.14 grounding 精度
- 状态：⬜ 未开始

## 目标
GUI 闭环（看家能力）依赖模型输出**准确的点击坐标**，而 VLM 的像素级 grounding 是公认难点。本任务解决两件事：(a) 坐标接地对齐到目标元素；(b) 执行后由下一帧感知校验是否生效，未生效则重试/换策略。

## 交付物
- [ ] `src/opensensorium/core/grounding.py`：坐标接地——把模型输出的坐标对齐到目标元素，必要时叠加元素检测/OCR/UI Automation 辅助定位。
- [ ] `src/opensensorium/core/verify_action.py`：执行后校验——用下一帧确认动作是否生效，未生效触发重试/换策略。
- [ ] grounding 精度评测脚本 `scripts/eval_grounding.py`：拿真实截图让 MiniCPM-o/JoyAI 输出"点击某按钮"坐标，量像素误差。
- [ ] 测试：`tests/test_grounding.py`（接地对齐逻辑、校验重试逻辑用 mock 帧）。

## 实现要点
- 这是 M2"看家能力"可靠性的关键一环（§12.14 与 §12.1 并列为 M2 关键风险）。
- 两款模型 grounding 精度需分别评测，差的那款更依赖 OCR/元素检测兜底。
- 校验闭环：看→动→**再看确认**→（未生效）重试，是感知-运动技能（T20）的基础。

## 分工（Agent 做什么 / 人做什么）
- **Agent**：实现接地、执行后校验、`eval_grounding.py` 评测脚本 + 测试。
- **人**：4090 在线时亲自看两款模型的 grounding 像素误差数据，决定是否需要更强的 OCR/元素检测辅助定位。

## 验收标准（可验证）
- [ ] `scripts/eval_grounding.py` 产出两款模型在一组真实截图上的**点击坐标像素误差**数据，写入报告。
- [ ] `tests/test_grounding.py`：接地能把偏差坐标吸附到目标元素；模拟"动作未生效"时触发重试。
- [ ] 集成：给定"点击某按钮"任务，接地+校验后成功率显著高于裸坐标（记录对比数字）。

> ⚠️ 需人工验证：grounding 评测是关键风险项，建议你亲自看误差数据决定是否需要更强的辅助定位。
