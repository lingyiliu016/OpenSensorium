# T17 · 分层记忆（工作/情景/语义/程序）+ 巩固 + 检索回流 + 隐私

- 里程碑：M2.8
- 依赖：T09
- PRD 对应：FR-7、FR-6.5、§3.8、§6.5
- 状态：⬜ 未开始

## 目标
一个常开、一直在看的智能体会产生海量经验；只有滚动工作记忆等于"金鱼脑"。建分层记忆 + 巩固，让它有连续的自我，并满足隐私责任（保留期/脱敏/可删）。

## 交付物
- [ ] `src/opensensorium/memory/working.py`：工作记忆（复用 T09，滚动短期上下文）。
- [ ] `src/opensensorium/memory/episodic.py`：情景记忆（会话/事件流，可回放，与审计 T14 共底座）。
- [ ] `src/opensensorium/memory/semantic.py`：语义/长期记忆（事实、用户偏好；向量检索 + 结构化存储）。
- [ ] `src/opensensorium/memory/procedural.py`：程序性记忆（"怎么做某任务"，喂 T20 Skills）。
- [ ] `src/opensensorium/memory/consolidation.py`：后台巩固——把工作记忆里值得留存的经验提炼进情景/语义。
- [ ] `src/opensensorium/memory/retrieval.py`：按当前情境检索相关记忆注入上下文（FR-7.3）。
- [ ] 隐私：保留期、脱敏、用户可查可删；敏感画面（密码框等）可配置不入记忆（FR-6.5）。
- [ ] 测试：`tests/test_memory.py`。

## 实现要点
- 向量存储可用轻量本地方案（如 sqlite + 向量索引/ chromadb），先跑通再优化。
- 巩固在后台异步进行，不阻塞主回路。

## 分工（Agent 做什么 / 人做什么）
- **Agent**：实现四层记忆、巩固、检索、隐私控制 + 测试。
- **人**：确认偏好跨会话生效、敏感画面不入记忆、删除后不再被检索到。

## 验收标准（可验证）
- [ ] `tests/test_memory.py`：写入→巩固→检索回流链路通；用户偏好可跨"会话"检索到。
- [ ] 隐私：标记敏感的画面/字段不进语义记忆；用户可删除某条记忆且检索不再命中。
- [ ] 集成：告诉它一个偏好，新会话里能被检索并影响行为。
