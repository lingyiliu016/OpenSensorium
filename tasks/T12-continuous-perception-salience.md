# T12 · 持续常开感知 + 显著性驱动主动发声

- 里程碑：M1
- 依赖：T09、T11
- PRD 对应：FR-2.2、FR-2.3、§3.7 感知触发
- 状态：⬜ 未开始

## 目标
让它像人一样"边看边聊、会主动开口"：持续常开感知，由**显著性**决定何时值得开口/行动，阈值可调（防话痨）。这一维对应主动性三触发源里的**感知触发**（§3.7）。

## 交付物
- [ ] `src/opensensorium/core/salience.py`：显著性评分（初期启发式：画面变化 + 语音停顿 + 任务进度）。
- [ ] 阈值可配；超阈值才产出主动 `Intent(say/act)`，否则 silent。
- [ ] 与两款模型的"是否发声"信号融合：MiniCPM-o 的 ~1Hz 自主决策 / JoyAI 的 `</silence>·</response>`（经 T03 归一），Core 侧再叠加显著性仲裁。
- [ ] 动态采样钩子：静止降帧、变化升帧（接 T04 背压）。
- [ ] 测试：`tests/test_salience.py`。

## 实现要点
- 不能只靠模型自身信号，也不能只靠 Core 启发式——两者融合（PRD §6.6 关键分歧 2 已指出映射关系）。
- 显著性阈值是"不烦人"的旋钮，M3 会进一步调优（T21）。

## 分工（Agent 做什么 / 人做什么）
- **Agent**：实现显著性评分、模型信号融合、动态采样 + 测试。
- **人**：长时间实测，把显著性阈值调到"主动但不烦"（两款各调一次）。

## 验收标准（可验证）
- [ ] `tests/test_salience.py`：静止画面→低分→silent；显著变化→高分→主动 Intent；阈值调高后主动频率下降。
- [ ] 集成：常开运行下，安静无变化时它不话痨；出现明显变化时主动开口（两款各验一次）。
- [ ] 动态采样：静止时帧率降、变化时升（日志可见）。

> ⚠️ 需人工验证："主动但不烦"的体感由你实测，阈值按体验调。
