# T14 · Safety 层：确认门 · 急停 · 审计回放 · 作用域

- 里程碑：M2
- 依赖：T13
- PRD 对应：FR-6、FR-3.5、§3.4、§8
- 状态：⬜ 未开始

## 目标
因为它真的会动手，误动作代价高。**一切动作/委派都必须过 Safety 层**。从第一天起就要有确认门、全局急停、审计回放、作用域边界。

## 交付物
- [ ] `src/opensensorium/safety/gate.py`：动作确认门——危险动作（删除、支付、发送、关机等）需人工确认或显式授权模式（FR-6.1）。
- [ ] `src/opensensorium/safety/kill_switch.py`：全局急停，热键/语音一键停止一切动作与生成（FR-6.2），**始终可用**。
- [ ] `src/opensensorium/safety/audit.py`：审计回放——记录"看到什么→决策→执行了什么动作/委派"，可回放（FR-6.3；与情景记忆共底座）。
- [ ] `src/opensensorium/safety/scope.py`：作用域边界，可限制到指定窗口/应用/目录（FR-6.4）。
- [ ] 危险动作白/黑名单可配。
- [ ] 把 T13 执行器、（未来）T16 委派统一接到本层。
- [ ] 测试：`tests/test_safety.py`。

## 实现要点
- 默认最小权限；危险动作默认需确认（NFR 安全）。
- 急停必须能在任何状态下中止（包括正在执行的动作与正在生成的输出）。
- 为物理执行器分级授权预留接口（车/机器人授权尤严，§9.4）。

## 分工（Agent 做什么 / 人做什么）
- **Agent**：实现确认门、急停、审计、作用域 + 测试。
- **人**：实测急停在真实运行中的即时性、确认门体验（安全关键，建议多审几轮）。

## 验收标准（可验证）
- [ ] `tests/test_safety.py`：危险动作被确认门拦截；急停触发后所有动作/生成停止；越作用域动作被拒；每个动作产生审计记录。
- [ ] 集成：一次"删除文件"意图必须先确认才执行；急停热键实测立即生效。
- [ ] 审计可回放一段"感知→决策→动作"序列。

> ⚠️ 需人工验证：急停在真实运行中的即时性、确认门体验由你实测（安全关键，建议多审几轮）。
